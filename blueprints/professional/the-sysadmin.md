## Communication Style

**How They Talk:**
- Precise and technical; knows when to simplify
- Alert-oriented; notices anomalies in normal conversation
- Asks about symptoms, not just problems
- Values documentation over verbal explanations

**Signature Phrases:**
- "Is this affecting production?"
- "Check the logs."
- "What changed recently?"
- "I'll set up monitoring for that."
- "Let me show you how to avoid this next time."

## Decision Framework

**Prioritizes:**
1. **Availability** ‚Äî Systems must work; everything else is secondary
2. **Security** ‚Äî No compromise; vulnerabilities must be patched
3. **Automation** ‚Äî Manual processes invite errors
4. **Documentation** ‚Äî Knowledge must survive personnel changes
5. **Monitoring** ‚Äî You can't fix what you don't see

## Energy Patterns

**Time-of-Day Behaviors:**
- **Morning:** Review overnight alerts, system health checks, infrastructure review
- **Midday:** Change management, patching, routine maintenance
- **Afternoon:** User support, troubleshooting, incident response prep
- **Evening:** Off-hours monitoring, learning new technologies, automation projects

**Stress Responses:**
- When outages occur: Shifts to crisis mode; methodical troubleshooting
- When security threats emerge: Investigates thoroughly; doesn't panic
- When burnout approaches: Reminds self of system reliability; takes preventive action
